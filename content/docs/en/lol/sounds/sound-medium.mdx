## Understanding Bank Structure

Riot splits audio into two separate files intentionally:

- `audio.bnk` / `audio.wpk`: contains the raw sound data only.
- `events.bnk`: contains the logic hierarchy (events, action sequences,
  routing).

`.wpk` is a Riot-original format. It can only hold audio files and has no
metadata header. Think of it as a lightweight container purely for sound data.

Wwise's hierarchy is not nested. Instead, every element carries an `ID` and a
`ParentID`. This flat structure makes it possible to inject new elements that
attach themselves to existing parents in the hierarchy, without needing to
replace the entire tree.

For example: you create a bus called `MUS` in your project, and under it a child
called `bus_aurecueil`. When your custom bank loads, the dummy `MUS` entry is
ignored (the real one is already loaded), and `bus_aurecueil` gets attached to
the original `MUS` automatically.

## Loading Order and Overrides

In League, **first loaded wins**. If two banks define the same element, the
second one is silently discarded.

Since core buses like `MUS` are loaded very early in the game's boot process, you
can't replace them directly. However, you _can_ override individual events by
making sure your custom bank loads before the original one.

You control this through the `.bin` file:

```text
bankPath: list[string] = {
    "ASSETS/Sounds/Wwise2016/VO/en_US/Characters/Yone/Skins/Base/Yone_Custom_VO_audio.bnk"  ← Your bank, loaded first
    "ASSETS/Sounds/Wwise2016/VO/en_US/Characters/Yone/Skins/Base/Yone_Base_VO_audio.bnk"
    "ASSETS/Sounds/Wwise2016/VO/en_US/Characters/Yone/Skins/Base/Yone_Base_VO_audio.wpk"
    "ASSETS/Sounds/Wwise2016/VO/en_US/Characters/Yone/Skins/Base/Yone_Base_VO_events.bnk"
}
```

Your custom bank overrides matching elements in the original events bank because
it's listed first.

**Note on IDs:**

- Events match by **name**, so as long as your event has the same name, it
  overrides correctly.
- Many other hierarchy elements (random containers, sequences, switches, etc.)
  use **random IDs generated by Wwise**. To override these, you need to manually
  match the original IDs in your `.wwu` project files.

### Audio Settings: Attenuation

Attenuation controls how sound behaves over distance. If you're adding new
sounds, always set this up explicitly, or they may sound wrong in-game (too loud
up close, audible from across the map, etc.). Distance is defined in in-game
units (you can use champion range as a reference for those units).

<img src="/wiki-images/wwise-pos.png" alt="Wwise positioning settings" />

<img src="/wiki-images/wwise-atten.png" alt="Wwise attenuation settings" />

**Always disable Height Spread.** League is played on a flat plane, and height
spread can cause unintended behavior.

### Audio Routing: Using the Right Bus

Every sound in Wwise routes through an output bus. Use the correct bus for your
sound type, or it won't be mixed properly in-game (wrong reverb, wrong volume
fader, etc.).

| Bus ID         | Purpose                                  |
| -------------- | ---------------------------------------- |
| `MUS`          | Music (wet, includes reverb/processing)  |
| `SFX`          | SFX (wet)                                |
| `daad0rhm`     | SFX (dry), **use this for champion SFX** |
| `SFX_HUD`      | Notifications & UI (wet)                 |
| `janep6uv`     | Announcer (dry)                          |
| `mus_lag_vp_0` | Champion VO (dry)                        |
| `sfx_npc`      | Ambience (wet)                           |
| `sfx_eog`      | End of Game (wet)                        |

When in doubt, prefer the **dry** version of the bus for your category. The wet
buses have their effects applied (for example, Evelynn distorts SFX during her
passive, so routing sounds through the `SFX` bus bypasses that distortion).
